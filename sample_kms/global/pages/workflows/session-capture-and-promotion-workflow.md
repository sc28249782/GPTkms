Use a three-step flow when preserving useful work into a long-term KMS:

1. capture the session as a raw source inside project scope
2. distill the reusable parts into project pages with clear titles and provenance
3. promote only clearly reusable knowledge into global scope

This keeps transcripts and evidence separate from compiled knowledge, while preserving a clean path from local discovery to reusable long-term memory.

The promotion bar should stay high. Most session knowledge belongs in project scope first.
