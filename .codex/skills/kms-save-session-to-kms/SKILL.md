# Kms Save Session To Kms

Use this skill to preserve the current session into project memory in a repeatable way.

Read [references/workflow.md](references/workflow.md) for the detailed capture sequence and promotion rules.

## Workflow

1. Distill the session into a concise summary.
2. Save that summary as a raw source in project scope.
3. Create or update the project pages that represent the durable knowledge.
4. Update the project index or log if new durable pages were added.
5. Promote only clearly reusable knowledge into global memory.

## Rules

- Save the full session knowledge to project scope first.
- Treat raw sources as evidence, not final memory.
- Prefer updating existing pages over creating redundant pages.
- Promote only reusable patterns, workflows, or preferences.
